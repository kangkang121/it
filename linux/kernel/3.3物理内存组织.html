              一、体系结构

              目前多处理系统重有2种体系结构：

              1）非一致内存访问（Non-Uniform Memory Access NUMA）：指向内存被划分成多个内存节点的处理器系统，访问一个内存节点花费的时间取决去处理和内存节点的距离。每个处理器都有一个本地内存节点，处理器访问本地内存节点的速度比访问其他内存节点的速度快。NUMA是中高端服务器的主流体系架构；

              2）对称处理器（Symmetric Multi-Processor SMP）：一致内存访问（Uniform Memory Access UMA），所有处理器访问内存花费的时间是相同的，每个处理器的地位实时平等的，仅在内核初始化的时候不平等，0号处理器作为引导处理器负责内核初始化，其他处理器等待内核初始化完成。

              在实际应用中可以采用混合体系结构，在NUMA节点内部使用SMP体系结构。

              二、内存模型

              内存模型是从处理器的角度看到的物理内存分布情况，内个管理不同内存模型的方式存在差异，内存管理子系统支持3种内存模型。

              1）平坦内存（Flat Memory）：内存的物理地址空间是连续的，没有空洞。

              2）不连续内存（Discontiguous Memory）：内存的物理地址空间存在空洞，这种模型可以高效的处理空洞。

              3）稀疏内存（Sparse Memory）：内存的物理地址空间存在空洞，如果支持内存热插拔，只能选择稀疏内存模型。

              什么情况会出现内存的物理地址空间存在空洞？系统包含多块物理内存，两块内存的物理地址空间之间存在空洞。一块内存的物理地址空间也可能存在空洞，可以查看处理器的参考手册获取分配给内存的物理地址空间。

              如果内存的物理地址空间是连续的，不连续的内存模型会产生额外的开销，降低性能，素以平坦内存模型是更好的选择。

              如果内存的物理地址空间存在空洞，应该选择哪种内存模型？

              平坦内存模型会为空洞分配page结构体，浪费内存；不连续内存模型对空洞做了优化处理，不会为空洞分配page结构体；针对平坦内存来说不连续内存是更好的选择。

              稀疏内存模型是实验性的，尽量不要选择稀释内存模型，除非内存的物理地址空间很稀疏或者支持内存热插拔，其他情况应该选择不连续内存模型。

              三、三级结构

              内存管理子系统使用节点（node）、区域（zone）、页（page）三级结构描述物理内存。

              节点node

              内存节点分为2种情况：

              1）NUMA系统的内存节点，根据处理器和内存的距离划分；

              2）在具有不连续的内存UMA系统中，表示比区域的级别更高的内存区域，根据物理地址是否连续划分，每块物理地址连续的内存是一个内存节点。

              如下图所示，内存节点使用pglist_data结构体描述内存布局，内核定义了宏NODE_DATA(nid)，它用来获取节点的pglist_data实例。对于平坦内存模型，只有一个pglist_data实例：contig_page_data。

              • 成员

                • node_id：内存区域数组；

                • nr_zones：是内存节点包含的内存区域的数量；

                • node_start_pfn：起始物理页号；

                • node_present_pages：实际存在的物理页总数；

                • node_spanned_pages：包含空洞的物理页总数；

                • node_mem_map：页描述符数组，每个物理页对应一个页描述符；注意：成员node_mem_map可能不是指向数组的第一个元素，因为页描述符页数组的大小必须对齐2的(MAX_ORDER - 1)次方，MAX_ORDER-1是页分配器可分配的最大阶数；

              • ZONE_DMA：DMA区域，“Direct Memoroy Access”直接内存访问，如果有些设备不能直接访问所有的内存，需要使用DMA区域，例如旧的工业标准体系结构（Inudeustry Standard Architecture ISA），总线只能直接访问16MB以下的内存；

              • ZONE_DMA32：DMA32区域，64位系统，如果既要支持直接访问16MB以下内存的设备，又要支持直接访问4GB以下内存的32位设备，那么必须使用DMA32区域；

              • ZONE_NORMAL：普通区域，直接映射到内核虚拟地址空间的内存区域，又称为线性映射区域，内核虚拟地址和物理地址是线性映射的关系，即虚拟地址 = （物理地址 + 常量）。是否需要使用页表映射？不同处理器实现不同，例如ARM处理器需要使用页表映射，而MIPS处理器不需要使用页表映射。

              • ZONE_HEGHMEM：高端区域，32位时代的产物，内核和用户地址空间按1:3划分，内核地址解空间只有1GB，不能把1GB以上的内存直接映射到内核地址空间，把不能直接映射的内存划分到高端内存区域，通常把DMA区域、DMA32区域和普通区域统称为低端内存区域。64位系统的内核虚拟地址空间非常大，不在需要高端内存区域。

              • ZONE_MOVABLE：可移动区域，它是一个伪内存区域，用来防止内存碎片；

              • ZONE_DEVICE：设备区域，为支持持久内存（persistent memory）热插拔增加的内存区域；

              物理页

              每个物理页对应一个page结构体，称为页描述符，内存节点的pglist_data实例的成员node_mem_map指向该内存节点包含的所有物流页的页描述符组成的数组。

              结构体page的成员flags的布局如下：

              1
              1
              | [SECTION] | [NODE] | [ZONE] | [LAST_CPUPID] | ... | [FLAGS]

              其中SECTION是稀疏内存模型中的段编号，NODE是节点编号，ZONE是区域类型，FLAGS是标志位。

              内联函数page_to_nid用来得到物理页所属地额内存节点的编号，page_zonenum用来得到哦呜立业所说的内存区域类型。
