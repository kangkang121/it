                        • 进程的退出

                        进程退出分为2种情况，进程主动退出和终止进程。

                        一、主动退出

                        系统调用函数

                        内核提供了两个使用进程主动退出的系统调用：

                        1. exit用来使一个线程退出，void exit(int status);

                        2. Linux私有系统调用exit_group用来结束一个线程组中的所有线程void exit_group(int status);

                        glibc库封装了库函数exit、_exit和_Exit用来让进程退出，这些库函数调用系统调用exit_group。库函数exit和_exit的区别是exit会执行由进程使用ateexit和one_exit注册的函数。

                        注意：使用的是glibc库的函数exit不是系统调用exit。

                        关注进程退出

                        当进程退出的时候，根据父进程是否关注退出事件处理上存在差异：

                        1. 如果父进程关注退出事件，子进程退出时释放各种资源，只留下一个空的进程描述符，变成僵尸进程，发送信号SIGCHLD通知父进程，父进程在查询终止的原因以后回收子进程的进程描述符；

                        2. 如果父进程不关注退出事件，子进程退出时释放各种资源，释放进程描述符，自动消失；

                        默认情况下，父进程时关注退出事件的，可以通过系统调用sigaction针对信号SIGCHLD设置标志SA_NOCLDWAIT来指示子进程退出时不要变成僵尸进程，或者设置忽略信号SIGCHLD。

                        查询子进程终止原因

                        Linux内核中提供了3个系统调用来等待子进程的状态改变，状态改变包括：子进程终止，信号SIGSTOP使子进程停止执行，或者喜好SIGCONT使子进程继续执行。3个系统调用：

                        1. pid_t_waitpid(pid_t pid, int *wstatus, int options);

                        2. int waitid(idtype_t idtype, id_t id, siginfo_t *infop, int options);

                        3. pid_t wait4(pid_t pid,int *wstatus, int options, struct rusage *rusage);

                        注意：wait4已经废弃，新的程序应该使用waitpid和waitid。

                        进程退出以后需要父进程回收进程描述符，如果父进程先退出，子进程称为“孤儿”，谁为子进程回收进程描述符呢？父进程退出时需要给子进程寻找一个“领养者”，按照下面的顺序选择领养“孤儿”的进程。

                        1. 如果进程属于一个线程组，且线程组还有其他线程，那么选择任意一个线程；

                        2. 选择最亲近的充当替补领养者的祖先进程，进程可以使用系统调用prctl(PR_SET_CHILD_SUBREAPER)把自己设置为”替补领养者（subreaper）“；

                        3. 选择进程所属的进程号命名空间中的1号进程；

                        线程组退出

                        系统调用exit_group实现线程组退出，执行流程如下：

                        1. 如果线程组正在退出，从信号结构体的成员group_exit_code取出退出码；

                        2. 如果线程组未处于正在退出的状态，并且线程组中的线程数大于等于2

                          1. 关闭终端申请锁；

                          2. 如果线程组此时正在退出，那么从信号结构体的成员group_exit_code取出退出码；

                          3. 如果线程组未处于正在退出状态，就会做以下处理：

                            1. 把退出码保存在信号结构体的成员group_exit_code中，传递给其他线程；

                            2. 给线程组设置正在退出的标志；

                            3. 向线程组的其他线程发送杀死信号，然后唤醒线程，让线程处理杀死信号；

                          4. 释放锁并开启中断；

                        3. 当前线程调用do_exit函数进行退出；

                        举例说明：假设一个线程组有两个线程线程1和线程2，线程1调用exit_group使线程组退出。

                        • 线程1的执行过程：

                          1. 把退出码保存在信号结构体成员group_exit_code中，传递给线程2；

                          2. 给线程组设置正在退出的标志；

                          3. 向线程2发送杀死信号，然后唤醒线程2，让线程2处理杀死信号；

                          4. 线程1调用函数do_exit退出；

                        • 线程2退出的执行流程如下：线程2返回用户模式的时候，发现收到了杀死信号，于是处理杀死信号，调用do_group_exit函数：

                          1. 因为线程组正在处于退出的状态，所以线程2从信号结构体的成员group_exit_code取出退出码；

                          2. 线程2调用函数do_exit退出；

                        • 线程2有3中情况返回用户模式：

                          1. 执行完系统调用；

                          2. 被中断抢占，终端处理程序执行完；

                          3. 执行指令时发生异常，异常处理程序执行完；

                        do_exit执行流程：

                        1. 释放各种资源，把资源对应的数据结构的引用计数减1，如果引用计数变为0，就释放数据结构；

                        2. 调用exit_notify函数，先称为“孤儿”的子进程选择“领养者”，然后把自己的死讯告知父进程；

                        3. 把进程状态设置为死亡（TASK_DEAD）；

                        4. 最后一次调用函数__schedule以调度进程；

                          死亡进程最后一次调用函数__schedule调度进程时，进程调度器做了如下特殊的处理：

                        二、终止进程

                        系统调用函数

                        终止进程时通过给进程发送信号实现的，内核提供了发送信号的系统调用。

                        1. kill用来发送信号给进程或者进程组；int kill(pid_t pid, int sig);

                        2. tkill用来发送信号给进程，参数tid是线程标识符；int tkill(int tid, int sig);

                        3. tgkill用来发送信号给线程，参数tgid是线程组标识符，参数tid是线程标识符；int tgkill(int tgid, int sig);

                        tkill和tgkill时Linux私有的系统调用，tkill已经废弃被tgkill取代。

                        kill执行流程

                        系统调用kill（源文件kernel/signal.c）负责向线程或者线程组发送信号，执行流程如下图所示：

                        1. 如果pid> 0：调用kill_pid_info来向线程pid所属的线程组发送信号；

                        2. 如果pid= 0：向当前进程组发送信号；

                        3. 如果pid< -1：想组长标识符为-pid的进程组发送信号；

                        4. 如果pid= -1：向除了1号线程和当前线程组以外的所有线程组发送信号；

                        子进程终止原因

                        waitid

                        系统调用waitid的原型：

                        1
                        1
                        int waitid(idtype_t idtype, id_t id, siginfo_t *infop, int options);

                        • 参数说明：

                          1. idtype：指定标识符类型有以下几种枚举：

                            • P_ALL：表示等待任意子进程，忽略参数id；

                            • P_PID：表示等待进程位id的字进程；

                            • p_PGID：表示等待进程组标识符是id的任意子进程；

                          2. options：取值是0或者以下标志的组合：

                            • WEXITED：等待退出的子进程；

                            • WSTOPPED：等到收到信号SIGSTOP并停止执行的子进程；

                            • WCONTINUED：等到收到信号SIGCONT并继续执行的子进程；

                            • WNOHANG：如果没有子进程退出，立即返回；

                            • WNOWAIT：让子进程处于僵尸状态，以后可以再次查询状态信息；

                        waitpid

                        系统调用waitpid的原型：

                        1
                        1
                        pid waitpid(pid_t pid, int *wstatus, int options);

                        • 参数说明：

                          1. pid取值：

                            • 大于0：表示等待进程号pid的子进程；

                            • 等于0：表示等待和调用进程属于同一个进程组的任意子进程；

                            • 等于-1：表示等待任意子进程；

                            • 小于-1：表示等待进程组标识符是pid的绝对值的任意子进程；

                          2. options：取值是0或者以下标志的组合：

                            • WNOHANG：乳沟没有子进程退出，立即返回；

                            • WUNTRACED：如果子进程停止执行，但是不被ptrace跟踪，那么立即返回；

                            • WCONTNUED：等待收到信号SICONT并继续执行的子进程；

                            以下选项是Linux私有的，和使用clone创建子进程一起使用：

                            • __WCLONE：只等待克隆的子进程；

                            • __WALL：等待所有的子进程；

                            • __WNOTHREAD：不等待相同线程组中其他线程的子进程；

                        do_wait

                        系统调用waitpid、waittid和wait4把主要工作委托给函数do_wait，函数do_wait执行流程如下图所示：

                        遍历当前线程组的每个线程，针对每个线程遍历它的每个子进程，如果是僵尸进程，调用函数eligible_child来判断是否符合等待条件的子进程看，符合调用函数wait_task_zombie进行处理。

                        函数wait_task_zombie执行流程：

                        1. 如果调用者没有传入标志WEXITED，说明调用者不想等待退出的子进程，就直接返回；

                        2. 如果调用者传入标志WNOWAIT，表示调用者向塘子进程处于僵尸状态，以后可以再次查询子进程的状态信息，那么只读取进程的状态信息，从线程的成员exit_code读取退出码。

                        3. 如果调用者没有传入WNOWAIT标志，处理过程：

                          1. 读取进程的状态信息，如果线程组处于正在退出的状态，从线程组的信号结构体的成员group_exit_code读取退出码，如果只是一个线程退出，那么从线程的成员exit_code读取退出码；

                          2. 把状态切换到死亡，释放进程描述符；
